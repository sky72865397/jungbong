# 택배 주소 자동변환 도구 프로젝트

## 이 프로젝트가 하는 일
거래처에서 받은 택배 주문 문자(카카오톡/문자)를 붙여넣으면:
1. 주문자 / 휴대폰번호 / 품목명 / 박스수량을 자동 분리
2. 받는주소(지번주소)를 도로명주소(신주소)로 자동 변환 (juso.go.kr 검색API)
3. 정해진 엑셀 양식으로 바로 내보내기(.xlsx 다운로드)

비개발자(택배·이커머스 운영 담당자)가 매일 쓰는 실무 도구임. 코드보다 "실제로 작동하고 오배송이 안 나는 것"이 최우선.

## 핵심 파일
- `주소변환도구.html` : 메인 도구. HTML 한 파일에 전부 들어있음(외부 의존성은 CDN의 SheetJS만). 더블클릭으로 브라우저에서 바로 실행됨.
- `senders.json` : 보내는사람(거래처) 목록. 아래 형식.

```json
[
  { "name": "정규학", "phone": "010-5331-7365", "addr": "성검로 443" }
]
```

## juso.go.kr API 키 (이미 발급받음)
- 검색API 키(자동변환용, 이게 메인): `U01TX0FVVEgyMDI2MDYyMzIwNTQwOTExOTQ5NDA=`
- 팝업API 키: `U01TX0FVVEgyMDI2MDYyMzIwNTkxOTExOTQ5NDE=`
- 모바일팝업 키: `U01TX0FVVEgyMDI2MDYyMzIxMDEwNjExOTQ5NDI=`

검색API는 브라우저에서 직접 호출하면 CORS에 막히므로, **JSONP 방식**(`addrLinkApiJsonp.do` + `callback` 파라미터 + `resultType=json`)으로 호출해야 함. 이 부분이 이미 구현돼 있음. 절대 일반 fetch/axios로 바꾸지 말 것(timeout 남).

## 현재까지 구현된 기능
- 문자 파싱: 상단 품목줄("6/22 호랑이콩 택배 8키로 12건 13박스")에서 품목 자동 인식 → 전 건에 "호랑이콩 8키로"로 적용
- "이름(휴대폰)" 줄 + 다음 줄(들) = 그 사람 주소 (주소가 여러 줄로 쪼개져도 합침)
- 입금/메모("입금","최충경입금","(최은정입금)") 자동 제거
- 박스 수량: 숫자만 저장(예: "2박스"→"2"), 없으면 1
- 주소 변환 재시도: ①원본 ②아파트명 제거 ③도로명+번호만 추출(extractRoad, "온중로22"→"온중로 22", "830번길" 처리) ④맨끝 단어(상호) 떼기
- 변환 의심 자동판별(isSuspect): 원본 시/군/구 지명이 변환결과에 없으면 ⚠ 표시 + 분홍 배경. 정상 건은 안 봐도 됨
- 엑셀 내보내기: 양식 = 번호 / 보내는사람 / 보내는사람 휴대폰번호 / 보내는주소 / 주문자 / 휴대폰번호 / 품목명 / 박스 / 받는주소(도로명+상세)
- 파일명: 저장일 기준 `yymmdd-택배.xlsx` 자동
